# Procurely - AI-Powered Vendor Discovery Platform

## Project Overview
Procurely is a full-stack procurement search copilot that helps teams discover, evaluate, and compare vendors using AI-powered web search, evidence extraction, and scoring.

---

## Tech Stack

### Frontend (`/frontend`)
- **Framework**: React 18 + Vite 5 + TypeScript 5
- **Styling**: Tailwind CSS 3.4
- **State**: React Query + Context
- **UI**: Radix UI + Framer Motion + Recharts + Lucide Icons

### Backend (`/backend`)
- **Framework**: FastAPI + Python 3.11+
- **Database**: PostgreSQL/SQLite + SQLAlchemy 2.0 + Alembic
- **Auth**: JWT (python-jose)
- **AI**: OpenAI, Google Gemini
- **Search**: Serper, Tavily (parallel execution)
- **Scrape**: Jina Reader (FREE), Crawl4AI (self-hosted), httpx

---

## Development Rules

## Key Features

### 1. Procurement Request Flow
1. User creates request (title, category) → keywords auto-generated by LLM
2. Submit triggers pipeline: Expand → Parallel Search → Parallel Scrape → Extract → Dedup → Score → Logo
3. Results with real-time polling
4. Add to shortlists or generate reports

### 2. Vendor Scoring
- **Fit Score**: Matches must-have/nice-to-have criteria
- **Trust Score**: Source quality, evidence count, company completeness
- **Quality Score**: Research completeness and confidence
- **Overall**: `fit * 0.35 + trust * 0.25 + quality * 0.25 + price * 0.15`

### 3. AI Search (Procurement Chat)
- Conversational vendor search
- Database first → auto-triggers DeepResearch if < 3 vendors or avg score < 50
- Real-time progress with partial results

### 4. Two Search Systems
| System | Purpose | Speed |
|--------|---------|-------|
| AI Search | Quick lookup + auto web search | Instant to 2-3 min |
| Request Pipeline | Deep web discovery | 2-5 minutes |

---

## Pipeline Steps

1. **EXPAND** - LLM generates search queries
2. **PARALLEL_SEARCH** - Serper + Tavily simultaneously
3. **PARALLEL_SCRAPE** - Jina + Crawl4AI simultaneously (no URL dedup for richer data)
4. **EXTRACT** - LLM extracts vendor info (truncate 60KB)
5. **DEDUP** - Merge duplicates (name >85% similarity OR same domain)
6. **SCORE** - Calculate fit/trust/quality scores
7. **LOGO** - Fetch from Clearbit/favicon

## Multi-Provider System

### Search Providers
| Provider | API Key | Description |
|----------|---------|-------------|
| Serper | SEARCH_PROVIDER | Google Search (default) |
| Tavily | TAVILY | AI-optimized search |

### Scrape Providers
| Provider | Free | Description |
|----------|------|-------------|
| Jina Reader | Yes | LLM-ready markdown |
| Crawl4AI | Yes (self-hosted) | JS rendering |
| httpx | Yes | Fallback |

**Why no URL dedup?** Each provider returns different content for same URL. Keeping all gives richer data for LLM extraction.

### Bug Fixes & Stability (Jan 2026)

#### 1. ExtractedVendor `.get()` Error
- **Problem**: Pipeline failed with `'ExtractedVendor' object has no attribute 'get'`
- **Cause**: ExtractedVendor is dataclass (`.name`, `.data`, `.evidence`), not dict
- **Fix**: Convert to dict in `runner.py`, dual-support in `gap_analysis.py` using `hasattr()`

#### 2. Status Filter Not Working
- **Problem**: RUNNING/COMPLETED/FAILED filters didn't work
- **Cause**: These are `SearchRun.status`, not `ProcurementRequest.status`
- **Fix**: Added subquery join to filter by latest run status in `requests.py`

#### 3. Dashboard Stuck "1 Running"
- **Problem**: Dashboard showed active run that didn't exist
- **Fix**: Manually marked stuck run as FAILED

#### 4. HTML Report Keywords Bug
- **Problem**: Keywords showed as `[, ", l, a, p, t, o, p, ...]`
- **Cause**: JSON string iterated as characters instead of parsed
- **Fix**: Added `_parse_json_field()` helper, rewrote `reports.py` with proper JSON parsing and modern card design

**Key Learnings**:
- Always convert dataclasses to dict before passing to dict-expecting functions
- JSON fields can be string or already parsed - use safe parser
- `ProcurementRequest.status` ≠ `SearchRun.status` - filter accordingly

## Important Data Types

### ExtractedVendor (dataclass)
```
.name: str
.data: dict (pricing_model, description, etc.)
.evidence: list[dict]
.source_url: str
```

### ScoredVendor (wrapper)
```
.vendor: ExtractedVendor
.fit_score: float
.trust_score: float
```

**Always use `hasattr()` checks when handling these types.**
